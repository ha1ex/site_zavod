# ТЗ на лендинг — kaiten-mcp

> Сгенерировано фабрикой ТЗ (Контент-завод Кайтен) из сырых данных. Документ для ревью командой; машинный бриф — `content/briefs/kaiten-mcp.json`.

## Summary

Лендинг новой возможности Kaiten — MCP-сервер, который подключает ИИ-ассистентов (Claude, Cursor, VS Code и другие MCP-совместимые клиенты) к доскам и карточкам Kaiten. Аудитория — команды разработки и технические директора. Layout product-launch: сначала анонс и нарратив «зачем», затем возможности, шаги подключения (через токен доступа), доверие на фактах Kaiten и приглашение попробовать. Блок отзывов не используем — фича новая. Цель — try_free («Попробовать»). Подтверждено: подключение через URL + токен API, только облачная версия. Тарифную доступность и точный формат адреса сервера не подаём как факт (needs_confirmation).

## Аудитория

- **Основная:** Команды разработки: инженеры и тимлиды, которые ведут задачи в Kaiten и работают в ИИ-ассистенте и редакторе кода
- **Принимает решение:** Технический директор (CTO) или ИТ-директор — решает о подключении на команду; инициатор — сам разработчик
- **Роли:** Разработчик / инженер команды разработки, Тимлид команды разработки, Технический директор (CTO), ИТ-директор

**Боли:**
- Задачи ведутся в Kaiten, а работа идёт в ИИ-ассистенте и редакторе кода — статусы и карточки обновляются вручную
- Разработчик переключается между доской и рабочим окном, чтобы завести задачу или подвинуть статус
- Сводку по спринту приходится собирать руками, открывая доски и считая карточки
- Рутинные операции с таск-трекером отнимают время у инженеров

**Критерии выбора:**
- Работает с тем ИИ-инструментом, которым уже пользуется команда
- Открытый стандарт вместо самописных интеграций
- Данные остаются в контуре Kaiten, доступ под права команды
- Быстрое подключение без интегратора

**Возражения:**
- Не даст ли агент лишнего доступа к задачам
- Работает ли с нашим ИИ-клиентом
- На каком тарифе доступно
- Нужна ли доработка на нашей стороне

## Тексты блоков

### Hero
**ИИ-ассистент управляет задачами Kaiten**

Подключите Claude, Cursor или другой ИИ-инструмент к доскам Kaiten. Ассистент создаёт, читает и обновляет карточки по стандарту Model Context Protocol (MCP) — прямо из диалога, без переключения на доску.

### Мини-преимущества (BenefitsStrip)
Любой MCP-клиент · Карточки из диалога · Статусы без ручного клика · Сводки по спринтам

### Story (зачем это, MediaCopy)
**Меньше переключений между окнами**

Задачи живут в Kaiten, а работа идёт в ИИ-ассистенте и редакторе кода. Раньше статусы и карточки приходилось обновлять вручную, переходя между окнами. MCP-сервер убирает этот разрыв: ассистент сам обращается к доскам Kaiten и меняет задачи по вашему запросу.

### Возможности (FeatureGrid columns=3)
**Что умеет ассистент с доступом к Kaiten**

Читает доски и карточки: спросите статус спринта — получите сводку из Kaiten. Создаёт задачи из диалога: опишите словами — ассистент заведёт карточку с описанием и чек-листом. Обновляет статусы: агент двигает карточки по колонкам, пока вы остаётесь в редакторе кода. Собирает сводки: соберёт данные по спринту и вернёт результат. Работает по открытому стандарту: Model Context Protocol поддерживают Claude, Cursor, VS Code и другие клиенты. Доступ под права Kaiten: агент видит те доски, к которым открыт доступ.

### Как подключить (ProcessSteps)
**Подключение за несколько шагов**

1. Создайте токен доступа в Kaiten. 2. Укажите адрес MCP-сервера и токен в своём ИИ-клиенте. 3. Откройте агенту доступ к нужным доскам. 4. Управляйте задачами прямо из диалога.

### Доверие (StatStrip)
Kaiten на рынке с 2018 года · 100 000+ компаний · Данные хранятся в России · В реестре отечественного ПО

### Промо-приглашение (PromoBanner)
**Подключите ассистента к своим доскам**

MCP-сервер уже доступен. Попробуйте, как агент заводит задачи и двигает статусы за вас.

### Вопросы и ответы (FAQAccordion)
С какими ИИ-инструментами работает MCP-сервер — с любым MCP-совместимым клиентом: Claude, Cursor, VS Code и другими. Как подключить — создаёте токен доступа в Kaiten, указываете адрес сервера и токен в ИИ-клиенте. Что агент делает с задачами — читает доски, создаёт и обновляет карточки, двигает статусы, собирает сводки. Как настраивается доступ — агент работает под тем токеном и правами, которые вы ему открыли. Работает ли на локальной версии — сейчас MCP-сервер доступен в облачной версии Kaiten. Остаются ли данные в контуре Kaiten — да, обмен идёт с вашими досками.

### Финальный CTA (FinalCta)
**Отдайте рутину по задачам агенту**

Подключите MCP-сервер Kaiten к своему ИИ-ассистенту и управляйте задачами из диалога.
