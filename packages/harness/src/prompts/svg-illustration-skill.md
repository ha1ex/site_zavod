# SVG Hero Illustration ‚Äî TSX Skill

You generate **one TSX file** that exports a single React component matching the supplied `IllustrationSpec`. The output is consumed by a deterministic AST validator ‚Äî any deviation from the rules below fails the build.

## Output contract

Return ONLY a TSX file. No prose, no markdown fences, no explanations.

The file must have this exact shape:

```tsx
// AUTO-GENERATED by @buffalo/harness ‚Äî illustration <id>
import type { SVGProps } from 'react';

interface InnerProps extends SVGProps<SVGSVGElement> {
  dark: boolean;
}

function Inner({ dark, ...props }: InnerProps) {
  return (
    <svg
      viewBox="<x> <y> <w> <h>"
      xmlns="http://www.w3.org/2000/svg"
      role="img"
      aria-label="<short alt>"
      {...props}
    >
      {/* gradients, devices, decorations ‚Äî all colors expressed as `dark ? '#dark' : '#light'` */}
    </svg>
  );
}

export default function <PascalCaseId>(props: SVGProps<SVGSVGElement>) {
  return (
    <>
      <Inner dark={false} className="dark:hidden" {...props} />
      <Inner dark={true}  className="hidden dark:block" {...props} />
    </>
  );
}
```

## Hard rules (AST-enforced)

1. **One root `<svg>` per `Inner`** ‚Äî exactly one top-level SVG element inside the `Inner` function's return.
2. **All colors flow through the `dark` prop** ‚Äî every `fill`, `stroke`, `stop-color`, `flood-color` attribute must be either:
   - a JSX expression of the form `{dark ? '#xxxxxx' : '#yyyyyy'}`, or
   - `"none"` / `"currentColor"` (which are theme-agnostic), or
   - a `url(#id-d)` / `url(#id-l)` reference built via the same ternary, e.g. `fill={\`url(#glow-${dark ? 'd' : 'l'})\`}`.
   Never use a raw hex literal like `fill="#FF00AA"`.
3. **Every `id` ends with `-d` or `-l`** ‚Äî pick the suffix from the same `dark` ternary so the light and dark SVGs never clash in the DOM. Example:
   ```tsx
   <linearGradient id={`glow-${dark ? 'd' : 'l'}`}>‚Ä¶</linearGradient>
   ‚Ä¶
   <rect fill={`url(#glow-${dark ? 'd' : 'l'})`} />
   ```
4. **Digits inside `<text>` use `tabular-nums`** ‚Äî when an SVG text node contains a number (price, metric, chart label, percentage), add `className="tabular-nums"` (or `style={{ fontVariantNumeric: 'tabular-nums' }}` if `className` collides with another need).
5. **No `stopColor="transparent"`** ‚Äî use `stopOpacity={0}` on a real color stop. Reason: Safari renders `transparent` as opaque black on some gradients.
6. **Default export is the dual-render wrapper** ‚Äî pair of `<Inner dark={false} className="dark:hidden" />` and `<Inner dark={true} className="hidden dark:block" />`. No more, no fewer, no other layout around them.

## Soft rules (style)

- Pick palette HEXes from `spec.palette` (`accent`, `glowDark`, `glowLight`). Mix neutrals from Tailwind's slate scale for surfaces and strokes.
- For `scene: 'device_showcase'` ‚Äî draw a laptop and/or phone frame, screen filled with a UI mockup matching `uiHint` (dashboard / chart / form / table). Keep UI primitives geometric and minimal.
- For `scene: 'isometric_dashboard'` ‚Äî single floating card with widgets, slight perspective skew.
- For `scene: 'metaphor'` ‚Äî abstract composition; no devices.
- For `scene: 'data_viz'` ‚Äî large chart as the centerpiece.
- Glow: radial gradient at `glow.centerX,centerY` with `glow.radius`, color `glowDark`/`glowLight` per palette.
- Decorations: plant / blob / sparkle / logo ‚Äî each ‚â§ 2% of canvas area, scattered without overlapping devices.
- Use `viewBox` from the spec verbatim. Do not add a hard width/height (let consumer size it via className).

## Hard rules ‚Äî DO NOT

- Do not add `<defs><style>` blocks or any CSS inside the SVG.
- Do not use `<image href>` or external assets.
- Do not import anything beyond `'react'` (for `SVGProps`).
- Do not add `onClick`, animations, or `<script>`.
- Do not split the output into multiple files. One TSX file.

## Self-check before returning

Read your output once and verify:

- [ ] File contains exactly one `function Inner(`, one `export default function`.
- [ ] Every `id=` value uses the `${dark ? 'd' : 'l'}` ternary suffix.
- [ ] No raw hex literal appears as a `fill`/`stroke`/`stop-color` attribute value ‚Äî every color is a `{dark ? ... : ...}` expression or `"none"`/`"currentColor"`.
- [ ] No occurrence of `stopColor="transparent"` (case-insensitive).
- [ ] Every `<text>` node containing digits has `className="tabular-nums"` or equivalent.
- [ ] `export default` returns the dual-render fragment with the two className strings exactly: `dark:hidden` and `hidden dark:block`.

If any check fails, fix the output before returning.
