# Воркшоп: обучение команды «Контент-завод Кайтен»

Материалы для обучения нетехнической маркетинг-команды. Цель — показать, что команда
**сама может создавать программные продукты**: с помощью нейронок делать себе
приложения и инструменты, вести их в GitHub и публиковать в продакшен на Vercel.
Лендинги — первый практический полигон. Всё общение с агентом — **по-русски, без
команд и кода руками**.

Формат: **один большой воркшоп (~3 часа) → неделя практики → воркшоп-разбор**.

```
ДЗ-0 (подготовка) → Воркшоп (GitHub → лендинг → деплой) → неделя практики (ДЗ-1)
→ Воркшоп-разбор (прикладной: ваши лендинги, деплои, ошибки, как работать дальше)
```

## Как открыть презентацию

Откройте `program.html` в браузере — двойной клик или перетащите в окно. Сервер не
нужен, всё в одном файле.

Навигация: **→ / Space / ↓** — следующий слайд, **← / ↑** — предыдущий; точки справа —
переход к любому слайду; **Home / End** — в начало / в конец. Для показа разверните
браузер на весь экран.

## Принципы материалов

- **Никаких команд.** Везде — русские фразы агенту («покажи в браузере», «сохрани в main»,
  «задеплой на Vercel»).
- **Последовательные схемы.** Цепочка цели, конвейер, цикл улучшения, цепочка деплоя —
  всё горизонтальными схемами «слева направо».
- **Весь путь за один воркшоп.** GitHub → первое изменение → лендинг → починка правил →
  деплой. К концу у каждого — публичная ссылка на свой лендинг.
- **Учим чинить правила, а не лендинги.** Брак указывает на шаг конвейера → чините
  правило шага → все следующие лендинги собираются правильно. Для этого есть
  **универсальный промпт для правок**.
- **Разбор — по реальному материалу.** Следующий воркшоп собирается из того, что
  команда принесёт: лендинги, деплои, ошибки, правки правил.
