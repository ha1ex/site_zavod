# Продуктовые факты Kaiten — источник истины по позиционированию

> **Каскад:** этот файл выигрывает по фактам/тарифам/позиционированию. Не использовать продуктовые утверждения, которых здесь нет, как факт — выносить в `needs_confirmation`. Проверять по базе знаний (faq-ru.kaiten.site) и продуктовым страницам.

## 1. Что такое Kaiten

Kaiten — порядок в рабочих процессах. Гибкая платформа для управления рабочими процессами: помогает компаниям экономить ресурсы, повышать производительность и концентрироваться на главном.

Российский сервис для управления проектами, задачами и командами. На рынке с 2018 года, доверяют 100 000+ компаний (Сбер, S7 Airlines, X5 Group, Ростелеком, Норникель и др.). Включён в реестр отечественного ПО. Данные хранятся в России, доступна установка на собственный сервер.

| Проблема | Решение Kaiten |
|---|---|
| Непрозрачность и запутанность процессов | Работа всей компании видна на одном экране |
| Узкие места и низкая скорость | Повышение эффективности за счёт устранения ненужных шагов |
| Высокий уровень контроля | Контроль без микроменеджмента |

Может заменить несколько инструментов: управление задачами (аналог Jira, Trello, Wrike), проекты на диаграмме Ганта (GanttPro, MS Project), служба поддержки/сервис-деск для заявок (Jira Service Desk, ITSM 365), документы и база знаний (Notion, Google Docs).

## 2. На чём делаем акцент

- Визуализация рабочих процессов: канбан-доски, несколько досок на одном экране.
- Управление проектами: диаграмма Ганта, ресурсное планирование, портфель проектов.
- Канбан и скрам: WIP-лимиты, дорожки (свимлайны), любое число этапов, спринты и цели спринтов, отчёты.
- Иерархия задач: родительские/дочерние с неограниченной вложенностью.
- Автоматизация: автоматизации, правила на досках, запланированные задания.
- Документы: совместная работа, база знаний, публикация документов.
- Управление большими командами: детальные права доступа, кастомные роли, синхронизация с директорией.
- Аналитика: подробные отчёты, скрам/канбан-отчёты, учёт времени, анализ блокировок и узких мест.
- Модуль Service Desk: заявки внутренних и внешних заказчиков.
- Автоматический перенос данных: Trello, Jira, Asana, Notion, ClickUp + универсальный импорт через CSV/JSON.
- Коробочная и облачная версии. Интеграции, дополнения, открытый API. Мобильное приложение.

## 3. Тарифы

**Триал-период** — тестовый период тарифа Pro на 14 дней с доступом ко всем модулям и функциям.

- **Free** — все необходимые функции для простых рабочих процессов, задач и документов. **Без ограничений по сроку и количеству участников.**
- **Standard** — 2 любых модуля в стоимости + дочерние/родительские задачи, несколько досок на пространстве, запланированные задания.
- **Pro** — 6 любых модулей в стоимости + всё из Standard.
- **Enterprise** — для крупных компаний от 100 человек: все модули и функции в максимальном объёме, установка на свой сервер, индивидуальная настройка и внедрение под ключ.

## 4. Что НЕ соответствует позиционированию

| ❌ Неверно | ✅ Верно |
|---|---|
| Kaiten — сервис для небольших компаний | Подходит для крупных компаний; небольшие команды тоже могут использовать |
| Kaiten — трекер задач | Система управления проектами, задачами и командами / рабочими процессами |
| Бесплатный тариф — для маленьких компаний | Бесплатный тариф не ограничен по числу пользователей |
| Бесплатный тариф — урезанный/базовый | Имеет все возможности для простых процессов; во многом функциональнее Trello |
| Бесплатный тариф ограничен по сроку | По сроку ограничен только триал; Free — бессрочный |
| Модуль «Документы» | Документы — ключевая функция, бесплатно на всех тарифах; модули — платные доп. функции |

## 5. Как говорить про Agile/Kanban/Scrum

- **Agile** — собирательный термин. Избегаем его как факт, даём конкретику. Когда говорят «Agile-методологии», обычно имеют в виду канбан-метод или скрам-фреймворк.
- **Kanban / Канбан** — это **метод** (не фреймворк). Пишем «канбан-метод» или «Kanban метод». Не «канбан-фреймворк».
- **Scrum / Скрам** — это **фреймворк** (не метод). Пишем «скрам-фреймворк». Не «Scrum-метод».

| ❌ | ✅ |
|---|---|
| Эджайл, Аджайл | Agile |
| Scrum-метод | скрам-фреймворк |
| Канбан-фреймворк | канбан-метод, Kanban метод |

Правила написания канбан/скрам кириллицей на публичной поверхности — `wiki/references/anglicism-dictionary.md`.
