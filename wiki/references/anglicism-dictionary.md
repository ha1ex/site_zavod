# Словарь англицизмов Kaiten — источник истины по лексике (§10)

> **SSoT для языкового гейта.** Читаемое зеркало `anglicism-dictionary.json`. Машина (валидатор `landing-language`) читает JSON; человек читает MD. Правишь — правь оба файла.

## Правило

На **маркетинговой поверхности** (заголовки, подзаголовки, карточки, кнопки, подписи, alt-тексты, вопросы и ответы, SEO title/description, поля брифа `mainPain/mainPromise/cta`) англицизмы с понятным русским аналогом — заменяем русским словом или раскрываем русским вариантом с иностранным термином в скобках при первом употреблении.

Это **не** относится к коду, служебным полям, именам классов и техническим атрибутам — там термины допустимы.

## Исключения

1. **Названия сервисов, продуктов, товарных знаков** не переводим: Kaiten, Кайтен, Jira, Trello, Notion, YouTrack, Asana, ClickUp, Wrike, GanttPro, MS Project, ITSM 365, Telegram, Макс, GitLab, GitHub, Zapier, Slack, Google Docs.
2. **Канбан и скрам** пишем **кириллицей**. Латинские `Kanban`/`Scrum` — только как часть названия интеграции/документации/цитаты или рядом с «метод»/«фреймворк».
3. **Аббревиатуры** (HR, SLA, KPI, B2B, Agile, ERP) — при первом употреблении русская расшифровка, дальше можно аббревиатуру: отдел персонала (HR), соглашение об уровне обслуживания (SLA), ключевые показатели эффективности (KPI).

## Словарь замен (основное)

| Англицизм | Замена | Строгость |
|---|---|---|
| landing / лендинг | страница, промостраница, страница решения | error / warning |
| hero, hero-блок | первый экран | error |
| CTA, call to action | кнопка действия, призыв к действию | error |
| FAQ | вопросы и ответы | error |
| feature / фича | возможность, функция, инструмент | error |
| use case | сценарий использования, рабочий сценарий | error |
| workflow | рабочий процесс | error |
| deadline / дедлайн | срок | error |
| onboarding / онбординг | адаптация | error / warning |
| feedback | обратная связь | error |
| roadmap | дорожная карта | error |
| template | шаблон | error |
| custom fields | настраиваемые поля | error |
| self-hosted / on-premise | локальная версия, установка на серверы компании | error |
| функционал | функциональность | error |
| таск / таска | задача, карточка | error |
| dashboard / дашборд | панель отчётов, сводка | warning (допустим, если аудитория так говорит) |
| backlog / бэклог | очередь задач, список задач | warning (бэклог допустим для ИТ) |
| sprint | спринт (для ИТ/скрам ок), иначе рабочий цикл | warning |
| service desk | служба поддержки, сервис-деск | warning |

| enterprise | крупная компания, корпоративный клиент | warning |
| trial | тестовый период, триал-период | warning |
| stakeholder | заинтересованная сторона, заказчик | warning |
| колл | звонок, встреча, совещание | warning |

## Примеры

❌ «HR-команда ведёт onboarding кандидатов и собирает feedback после trial-периода».
✅ «Отдел персонала ведёт адаптацию сотрудников и собирает обратную связь после испытательного срока».

❌ «Настройте workflow, добавьте custom fields и следите за dashboard».
✅ «Настройте рабочий процесс, добавьте настраиваемые поля и следите за показателями на панели отчётов».

❌ «Kanban и Scrum-команды видят backlog и sprint status».
✅ «Канбан- и скрам-команды видят очередь задач и статус рабочего цикла».
