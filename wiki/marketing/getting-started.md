# Маркетинг — начало работы с Контент-завод Кайтен

> Этот документ — пошаговая инструкция для маркетинговой команды. **Не требует знания кода.** Всё, что нужно — терминал, любой ассистент (Claude Code / Codex CLI / Cursor / Gemini) и понимание собственного продукта.

## Что вы получите за 5 минут

Лендинг в фирменном стиле Kaiten, собранный **только из разрешённых компонентов**, прошедший автоматические проверки на:

- бренд-голос (не пропустит «революционный», «лидер рынка», «AI-powered» как пустые слова),
- бизнес-правила (один hero, корректные CTA),
- domain-fit (карточка тикета не попадёт в CRM-лендинг),
- разнообразие (одинаковые блоки подряд блокируются).

На выходе — рабочий preview и ZIP-пакет, который команда фронта принимает как обычный PR.

## Один раз — установка

```bash
pnpm install
```

После этого репо готов. API-ключи не нужны — ассистент сам генерирует LandingSpec.

## Каждый раз — пять шагов

### 1. Решите, в каком домене ваш продукт

Открыть [`../references/domain-mock-matrix.md`](../references/domain-mock-matrix.md), выбрать один из 9 покрытых доменов: **PM, Support, CRM, HR, Marketing, BPM, Finance, E-commerce, Docs**.

Если ваш продукт ни в один не подходит (например, Healthcare или Education) — пишите в инженерную команду: «нужно создать 5-8 mock'ов для домена X». Это занимает 1-2 дня и решает проблему «навсегда».

### 2. Напишите brief

Создайте файл `content/briefs/<slug>.json`. Skeleton:

```json
{
  "product": "Опишите продукт одним предложением",
  "audience": ["роль 1", "роль 2"],
  "market": "B2B SaaS",
  "primaryGoal": "book_demo",
  "mainPain": "Что болит у клиента",
  "mainPromise": "Что вы обещаете",
  "proofPoints": ["числовой факт", "сертификат", "клиентский кейс"],
  "tone": "clear, professional, non-hype",
  "cta": "Получить демо",
  "pageArchetype": "saas",
  "pageLayout": "enterprise-modular-saas"
}
```

Подробное описание каждого поля — [`brief-fields.md`](brief-fields.md).

Где взять `slug`: короткий `kebab-case`, например `kaiten-crm` или `support-helpdesk`. Это будет URL — `/landings/<slug>`.

### 3. Запустите harness

Скажите вашему ассистенту:

> Сгенерируй лендинг по `content/briefs/<your-slug>.json`, slug `<your-slug>`.

Ассистент сам выполнит:

```bash
pnpm -w run harness agent build landing --slug <slug> --brief content/briefs/<slug>.json
```

Harness:
1. Резолвит ваш домен и проверяет покрытие mock-набором (гейт).
2. Ведёт сборку по фазам P0–P8, запрашивая ассистента на каждой LLM-фазе.
3. Валидирует каждую фазу, затем финальный пакет проверок.
4. Рендерит TSX, сохраняет.

Если выскакивает `manual-creation-required` — см. [`troubleshooting.md`](troubleshooting.md) и шаг 1.

### 4. Посмотрите preview

```bash
pnpm dev
```

Откройте `http://localhost:3000/landings/<your-slug>`.

Если что-то не так — переходите к [`edit-recipes.md`](edit-recipes.md): «как поменять заголовок», «как заменить картинку», «как переставить секции».

### 5. Approve

`http://localhost:3000/approve/<your-slug>` — слева preview, справа форма с реквизитами approval'а. Состояние пишется в `content/approvals/<slug>.json`. Approve не обязателен для просмотра, но обязателен для handoff.

### 5+. Handoff (для фронта)

Когда всё нравится:

```bash
pnpm -w run harness handoff <slug> --require-approved
```

В папке `out/` появится `landing-<slug>.zip`. Это самодостаточный пакет — отдайте фронту.

## Что вам помогает в работе

### Визуальный каталог всех блоков

```bash
pnpm --filter @kaiten/ui storybook
```

Откроется `http://localhost:6006`. Слева:
- `Landing/*` — все 22 секции с примерами props.
- `Mocks/_Catalog/AllVariants` — все 39 mock-вариантов сгруппированы по доменам.

Используйте каталог, чтобы **до генерации** понять «как выглядит pm-board», «чем отличается analytics-kpi от crm-analytics».

### Ассистент — ваш напарник

Любой ассистент (Claude Code / Codex / Cursor / Gemini) умеет работать с этим репо одинаково — через `AGENTS.md` и `SKILL.md` в корне. Это **открытый стандарт**, принятый Anthropic и OpenAI с конца 2025 года.

Скажите ассистенту:

> Прочитай SKILL.md в корне и помоги мне с лендингом про X.

Он сам найдёт инструкции и поведёт за руку.

## Частые вопросы

**Сколько времени занимает генерация?**
Обычно 10-20 минут: конвейер фазовый (P0–P8), на каждой фазе своя валидация. Чем полнее бриф, тем быстрее проходят фазы.

**Можно ли работать с разными ассистентами?**
Да. AGENTS.md + SKILL.md — кросс-инструментальные. Главное, чтобы ассистент имел доступ к файловой системе репо.

**Можно ли отредактировать готовый лендинг без перегенерации?**
Да — через ручную правку `content/landings/<slug>.json` + повторный `harness agent apply`. См. [`edit-recipes.md`](edit-recipes.md).

**Что если домена нет в матрице?**
Pipeline отказывается работать и даёт todo-список mock'ов к созданию. Это инженерная задача — пишите в команду фронта.

**Когда появится UI-редактор?**
Веха M3 плана — Puck-редактор на `/edit/<slug>` для редактирования копии / порядка секций / mock'ов без правки JSON. См. план в `.claude/plans/`.
