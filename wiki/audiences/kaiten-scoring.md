# Kaiten — скоринг целевой аудитории (B-сегмент, 20–250 лицензий)

> **SSoT для audience-score gate.** Этот документ — читаемое зеркало `kaiten-scoring.json`. Машина читает JSON; человек читает MD. Если правишь — правь оба файла.

## Зачем

Buffalo-harness перед тем, как отдать лендинг на превью, проверяет: **закрывает ли он приоритетные User Stories для приоритетных портретов ЛПР приоритетных сегментов?** Если score < 70 или падает must-pass правило — лендинг возвращается на ещё одну итерацию доработки.

Реализация: `packages/harness/src/validators/landing-audience.ts` + интеграция в `agent/ingest-landing.ts`. Документация флоу — `.claude/skills/buffalo-generate/SKILL.md` (шаг 4½).

## Формула

```
Audience-Adjusted Score = 0.4 × S1 + 0.3 × S2 + 0.2 × S3 + 0.1 × S4
```

| Субскор | Что меряет |
|---|---|
| **S1. Story coverage** | Сколько top-N приоритетных user stories покрыто секциями/копи/CTA лендинга. |
| **S2. Segment fit** | Насколько hero/features/FAQ упоминают приоритетные для брифа сегменты (взвешено по WIM). |
| **S3. Role addressability** | Покрыты ли две первые роли (Тимлид/PM + ЛПР) явным контентом. |
| **S4. CTA alignment** | Совпадает ли тип CTA с рекомендованным по WIM (Trial / Demo / PDF / Partner). |

**Гейт по умолчанию:** `score ≥ 70` И все must-pass green. Бюджет итераций — 3.

## Must-pass правила (любое падение → fail независимо от score)

1. **IT нуждается в сравнении или Trial.** Если сегмент включает IT — должна быть покрыта Story #1 (сравнение) или #3 (переезд с Jira), либо hero.primaryCta = Trial.
2. **Финансы / Госсектор нуждаются в security + Demo.** Story #5 (безопасность/on-prem/ГОСТ) должна быть покрыта, primary CTA включает Demo (не только Trial).
3. **SEO таргетит приоритет.** `seo.title + seo.description` упоминают хотя бы один резолвленный сегмент или ключевое слово из top-3 stories.
4. **Антипрофиль не primary.** Resolved segments не равны исключительно 'Антипрофиль'.

## Приоритеты для информационной архитектуры

- **P0 (критично):** сравнительные страницы; секция «Решения по отраслям» на главной.
- **P1 (критично):** лендинг «Безопасность & On-prem»; лендинг «Госсектор & Партнёры»; двойной CTA (Trial + Скачать PDF).
- **P2 (высокий):** страница миграции с Jira; видео-демо по сценариям.
- **P3 (средний):** убрать «Обучение процессам» с главной — в блог.
